[POST] /agreements/{agreement_id}/updates

Enables DSP to update agreement status to Approved or Rejected or Deleted or update account nickname. Nickname must be supplied when agreement status is Approved.

  • Update Status - status can be updated from in progress to approved or rejected, when it is approved nickname has to be provided.
  • Update Nickname - agreementId and nickname has be to supplied.
  • Delete Agreement - status can be updated from approved to deleted.

Request Header Dictionary

Parameter Name Required? Description
X-Request-ID Yes Each API request has an associated request identifier. It can be used to match the response to the request and used for tracing, troubleshooting and analysis purposes.
X-Product-ID Yes Unique identifier assigned to the product by Mastercard and given to the participant.
X-Participant-ID Yes Unique identifier assigned to the participant by Mastercard during onboarding.
X-JWS-Signature Yes JWS detached signature is applied to ensure data integrity.
Idempotency-Key Yes Unique identifier generated by the participant. Mastercard uses this identifier to recognize subsequent retries of the same request and ensure idempotent behaviour by sending the same response without repeating the operation again.
agreement_id Yes Unique identifier assigned by Mastercard, to the agreement, to be used in Payment Requests for routing them to the destined participant.
